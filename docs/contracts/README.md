# Contracts (계약 경계)

프로젝트의 공식 계약 경계 문서를 두는 위치입니다.

계약 경계에는 API 스키마, DB 스키마, 이벤트 포맷, public interface, 모듈 경계, 인증/권한 경계가 포함됩니다. 계약 경계면의 추가, 삭제, 이름 변경, 의미 변경, 호환성 변경은 사용자 확인 후 진행합니다.

## Naming (이름 규칙)

계약 문서는 압축적이고 순서가 유지되는 편이 유리하므로 다음 형식을 사용합니다.

```text
001-short-title.md
```

순번은 디렉토리 내에서 증가시키고, 제목은 짧은 kebab-case로 작성합니다.

## Style (작성 방식)

계약 문서는 문서마다 필요한 표현이 다를 수 있으므로 고정 템플릿을 강제하지 않습니다.

다만 다음 내용은 가능한 한 명확히 적습니다.

- 계약이 적용되는 경계
- 현재 형태와 책임
- 변경 시 사용자 확인이 필요한 조건
- 호환성, 마이그레이션, 외부 소비자 영향
- 변경 검증 방법

## Documents (문서 목록)

현재 등록된 계약 문서는 없습니다.
