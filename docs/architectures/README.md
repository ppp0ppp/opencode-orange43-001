# Architectures (아키텍처)

프로젝트의 공식 아키텍처 문서를 두는 위치입니다.

아키텍처 문서는 시스템 구조, 모듈 구조, 운영 구조, 기술 선택, 주요 설계 판단을 다룹니다. 확정된 판단은 별도 decisions 디렉토리로 분리하지 않고 이 디렉토리의 문서 또는 관련 공식 문서에 흡수합니다.

## Naming (이름 규칙)

아키텍처 문서는 압축적이고 순서가 유지되는 편이 유리하므로 다음 형식을 사용합니다.

```text
001-short-title.md
```

순번은 디렉토리 내에서 증가시키고, 제목은 짧은 kebab-case로 작성합니다.

## Style (작성 방식)

아키텍처 문서는 문서마다 필요한 표현이 다를 수 있으므로 고정 템플릿을 강제하지 않습니다.

다만 다음 내용은 가능한 한 명확히 적습니다.

- 다루는 시스템 또는 모듈 범위
- 현재 구조와 주요 책임
- 선택한 설계와 선택하지 않은 대안
- 운영, 보안, 비용, 성능 영향
- 관련 계약, 계획, 논의, 보고서 링크

## Documents (문서 목록)

현재 등록된 아키텍처 문서는 없습니다.
