# Reports

분석 보고서, 검토 결과, 도입 전략, 실험 결과를 Markdown으로 기록하는 위치입니다.

## Naming

보고서 파일은 다음 형식을 사용합니다.

```text
001-short-title.md
```

순번은 디렉토리 내에서 증가시키고, 제목은 짧은 kebab-case로 작성합니다.

## Usage

- 사용자가 판단해야 하는 큰 맥락은 보고서로 정리합니다.
- 외부 도구, plugin, skill, harness를 도입하기 전에는 검토 보고서를 남깁니다.
- 실험 결과는 결론, 관찰, 리스크, 다음 단계 중심으로 작성합니다.
- 장기적으로 확정된 결정은 `../decisions/`로 별도 기록합니다.
- 계약 경계면과 관련된 내용은 필요 시 `../contracts/`에도 반영합니다.

## Template

```md
# Report Title

작성일: YYYY-MM-DD

## 결론

## 배경

## 관찰

## 리스크

## 대안

## 권장 다음 단계
```
