# Reports (보고서)

분석 보고서, 검토 결과, 도입 전략, 실험 결과를 Markdown으로 기록하는 위치입니다.

보고서는 특정 시점의 관찰과 근거입니다. 보고서는 현재 실행 기준이 아니며, 기본 참조 대상도 아닙니다.

## Naming

보고서 파일은 다음 형식을 사용합니다.

```text
001-short-title.md
```

순번은 디렉토리 내에서 증가시키고, 제목은 짧은 kebab-case로 작성합니다.

## Usage

- 사용자가 판단해야 하는 큰 맥락은 보고서로 정리합니다.
- 외부 도구, plugin, skill, harness를 도입하기 전에는 검토 보고서를 남깁니다.
- 실험 결과는 결론, 관찰, 리스크, 다음 단계 중심으로 작성합니다.
- 장기적으로 확정된 판단은 `docs/contracts/`, `docs/architectures/`, `docs/plans/` 또는 관련 README에 흡수합니다.
- 계약 경계면과 관련된 내용은 필요 시 `docs/contracts/`에도 반영합니다.
- 작업자가 보고서를 참조해야 한다고 판단하면 참조 이유를 사용자에게 설명하고 승인받은 뒤 확인합니다.
- 계속 필요한 운영 기준, 설치 기준, 정책은 보고서에 남겨두지 않고 `README.md`, `PROJECT.md`, `AGENTS.md`, `docs/`로 승격합니다.

## Template

```md
# Report Title (보고서 제목)

작성일: YYYY-MM-DD

## 결론

사용자가 먼저 알아야 할 핵심 결론을 적습니다.

## 배경

조사나 보고서가 필요한 이유를 적습니다.

## 관찰

확인한 사실, 코드 위치, 실험 결과를 적습니다.

## 리스크

기술적, 운영적, 보안적, 비용상 리스크를 적습니다.

## 대안

가능한 선택지와 장단점을 적습니다.

## 권장 다음 단계

바로 실행할 수 있는 후속 작업을 적습니다.
```
